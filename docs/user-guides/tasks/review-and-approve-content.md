## When To Use This

Use this workflow when content has reached the approval stage and is waiting for your decision.

## Before You Start

- You need approver access.
- The item should already be prepared by the editor or planner.

## Steps

1. Open the approvals queue.
2. Review the item card for:
   - date
   - asset type
   - status detail
   - requester or author
   - approvers
   - checklist items
3. Read any warning or blocker message shown before approving.
4. Select `Open detail` if you need a closer review.
5. Select `Mark approved` only when the item is ready to proceed.

## What Happens Next

The item leaves the outstanding approvals queue and continues through the workflow toward publication.

## Common Problems

### The item has blockers

Do not approve it until those blockers are understood or resolved.

### You need changes but the queue only shows approve/open actions

Open the detail view and follow the team’s normal feedback process there.
