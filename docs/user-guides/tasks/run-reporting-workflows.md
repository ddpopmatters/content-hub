# Run Reporting Workflows

**Audience:** Content team members, admins  
**Owner:** TBD  
**Last reviewed:** 2026-03-30  
**Related guides:** `../overview.md`, `../quick-start.md`

## When To Use This

Use this workflow when you need formal reporting outputs rather than day-to-day planning views.

## Before You Start

- Make sure the underlying content data is up to date.

## Steps

1. Open `Reporting` from the sidebar for reporting workspaces and outputs.
2. Open `Insights` when you need the lighter analytics and engagement views.
3. Use the reporting workspace to review the current reporting period and required metrics.
4. Use analytics or engagement views when you need supporting performance context.

## What Happens Next

Reporting outputs can be shared with the rest of the team once the reporting period and metrics are complete.

## Common Problems

### You cannot find the right reporting area

Use `Reporting` for formal reporting workflows and `Insights` for lighter analysis views.
