## When To Use This

Use this workflow when you need to turn a reactive signal into owned response work with a deadline and a clear handoff into content production.

Rapid responses are useful for:

- reactive moments
- pre-bunks
- counter-disinformation work
- tightly timed response content linked to an opportunity

## Before You Start

Have the key response information ready:

- response title
- owner
- current status
- response mode
- trigger date
- due date and time

Optional but useful:

- sign-off route
- source opportunity
- linked entry
- campaign
- content pillar
- target platforms
- notes

## Steps

1. Open `Content`.
2. Switch to `Responses`.
3. Select `Create rapid response` if you are creating a new item.
4. Enter the response title.
5. Set the owner and current status.
6. Set the response mode and trigger date.
7. Enter the `Due at` deadline.
8. Add a sign-off route if the response needs a defined approval path.
9. Link a source opportunity or existing entry if the response came from another signal or is already tied to content.
10. Save the rapid response.

## What Happens Next

The rapid response becomes a tracked operational item. Once it is ready to become content, you can hand it into the normal content workflow and move the resulting entry through review and publication.

## Common Problems

### The response exists but is not tied to content yet

That is acceptable at first, but link it to an entry once production starts so the handoff is visible.

### The deadline is unclear

Do not leave `Due at` vague for reactive work. Rapid responses need a clear completion point.
