# Move Work Through Kanban

**Audience:** Content editors and planners  
**Owner:** TBD  
**Last reviewed:** 2026-03-30  
**Related guides:** `plan-content-in-calendar.md`, `review-and-approve-content.md`

## When To Use This

Use this workflow when a content item needs to move from drafting to review, approval, and publication.

## Before You Start

- Confirm the item is in the correct workspace.
- Make sure the entry details are complete enough for the next status.

## Steps

1. Open the relevant content item.
2. Review the current status.
3. Move the item through the standard workflow:
   1. `Draft`
   2. `Ready for Review`
   3. `Approved`
   4. `Published`
4. Stop at `Ready for Review` when the item needs approver attention.
5. Only move to `Published` when the item has genuinely gone live.

## What Happens Next

The updated status affects how the item appears in the production workflow and what the rest of the team expects to happen next.

## Common Problems

### The item looks ready but should not be approved yet

Keep it in `Draft` or `Ready for Review` until blockers are cleared.

### The status names do not match an older process

Content Hub uses the simplified four-stage workflow, even if older content used more detailed legacy labels.
