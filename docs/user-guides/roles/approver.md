# Approver Role

**Audience:** Content approvers  
**Owner:** TBD  
**Last reviewed:** 2026-03-30  
**Related guides:** `../tasks/review-and-approve-content.md`, `../troubleshooting.md`

## What This Role Usually Does

- review content assigned for approval
- check blockers before approving
- open item detail when a closer review is needed
- mark items approved when they are ready to proceed

## What To Watch For

- incomplete checklists
- missing context or supporting detail
- workflow blockers highlighted in the approvals card
- incorrect status or missing approvers

## Main Work Area

The approvals queue shows:

- items waiting for your approval
- key metadata such as date, asset type, and status detail
- action buttons for `Mark approved` and `Open detail`

## Handoffs

- if the item is ready, approve it
- if it is not ready, reopen the detail and follow the team’s feedback path before approving
